18. WORKFLOW UTAMA

Writing Flow

1. user input prompt
2. AI generate
3. user edit
4. brain.learn(revision)
5. sync ke GitHub

---

Chat Flow

1. user chat
2. AI streaming response
3. auto save session
4. summary update (jika panjang)

---

Learning Flow

1. detect event (revision / chat / raw writing)
2. extract pattern
3. update:
   - style_profile
   - thinking_profile
   - rules
   - memory

---

Reference Flow (Semi Auto)

1. AI detect need reference
2. ask user permission
3. if yes:
   - search web
   - summarize
   - store in references

---

19. Update v1.1 (API Changes)

- **Limitless Input**: Tidak ada batasan `max_length` pada endpoint `/chat/send`, `/ai/generate`, `/brain/learn/raw-writing`, dan endpoint terkait lainnya.
- **Error Handling Stream**: API tetap menggunakan _Streaming text chunks_, namun sekarang frontend menangani interupsi stream secara _graceful_ tanpa memunculkan error text palsu.

---

20. Update v1.2 (Data Sync API)

- Endpoint baru: `POST /api/import` menerima payload `UploadFile` (format ZIP hasil `/api/export`).
- Backend memproses ZIP, memeriksa list workspace, menghindari konflik `id`, dan menyimpan file import menjadi workspace independen.
