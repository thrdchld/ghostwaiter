# Ghostwriter Data Schema Standard

Dokumen ini menjelaskan standar struktur data untuk aplikasi Ghostwriter. Seluruh struktur direktori dan file JSON ini dipertahankan secara utuh 1:1 saat dilakukan sinkronisasi Push maupun Pull ke/dari repositori GitHub.

Struktur ini memastikan bahwa backup bersifat *future-proof*, modular, dan ramah pembaca ketika ditinjau di GitHub.

## Struktur Direktori Root

Saat disinkronisasi ke GitHub, file akan langsung berada di root directory repositori dengan struktur berikut:

```text
/
├── system/
│   ├── settings.json
│   └── workspaces.json
│
├── [workspace_id]/
│   ├── brain/
│   │   ├── style_profile.json
│   │   ├── thinking_profile.json
│   │   ├── memory.json
│   │   ├── rules.json
│   │   ├── conversation_memory.json
│   │   └── learning_proposals.json
│   │
│   ├── drafts/
│   │   ├── [draft_id].json
│   │   └── ...
│   │
│   └── learning/
│       ├── raw_writing/
│       │   ├── [raw_id].json
│       │   └── ...
│       └── revision_pairs/
│           ├── [revision_id].json
│           └── ...
```

## Penjelasan Komponen

### 1. System (`/system/`)
Menyimpan konfigurasi global dan pengaturan level aplikasi yang berlaku untuk seluruh workspace.
- `settings.json`: Konfigurasi API key, GitHub token, preferensi tema, provider AI (OpenRouter, Groq, Gemini, DeepSeek, Mistral, dll).
- `workspaces.json`: *Registry* seluruh workspace yang ada (id dan nama workspace).

### 2. Workspace Brain (`/[workspace_id]/brain/`)
Menyimpan inti kepribadian dan pengetahuan AI untuk workspace tersebut. File-file ini diperbarui secara berkala melalui usulan (*proposals*) maupun entri manual.
- `style_profile.json`: Profil aturan gaya bahasa.
- `thinking_profile.json`: Pola dan kerangka pikir AI.
- `memory.json`: Informasi kontekstual yang diingat secara permanen.
- `rules.json`: Aturan mutlak tambahan yang harus diikuti AI.
- `conversation_memory.json`: Ingatan spesifik tentang percakapan yang pernah terjadi.
- `learning_proposals.json`: Antrean usulan pembelajaran (*pending*, *approved*, *rejected*).

### 3. Drafts (`/[workspace_id]/drafts/`)
Menyimpan draf tulisan atau kode yang dikerjakan pengguna. Tiap draf disimpan sebagai satu file JSON independen agar meminimalkan konflik antar draf (ID draf bertindak sebagai nama file).

### 4. Learning Data (`/[workspace_id]/learning/`)
Menyimpan data sampel (*ground truth*) yang digunakan untuk mengajari AI.
- `raw_writing/`: Sampel teks tulisan manual dari pengguna.
- `revision_pairs/`: Pasangan teks asli (sebelum) dan revisi (sesudah) untuk ekstraksi pola gaya dan pemikiran.

## Standar Sinkronisasi (GitHub Push & Pull)

Ghostwriter menggunakan pendekatan sinkronisasi langsung (manual) untuk menghindari kompleksitas penggabungan (*merge conflict*) berbasis *timestamp*.

- **Push (Kirim ke GitHub)**: Membaca seluruh file `.json` sesuai hierarki di atas, membuat `Tree` baru melalui GitHub Git Database API, dan membuat satu *Commit* utuh. Seluruh isi repositori akan **ditimpa** agar sesuai dengan data lokal.
- **Pull (Ambil dari GitHub)**: Membaca `Tree` terbaru dari cabang `main` (atau `master`) di GitHub, lalu mengunduh dan **menimpa** file-file lokal. File lokal ekstra yang tidak ada di GitHub *tidak akan* dihapus (bersifat restorasi aman).
