GhostWriter Data Schema v1.0

1. Prinsip Data

Semua data GhostWriter berbasis:

- JSON structured storage
- Workspace isolation
- Versioned schema
- GitHub sync compatible
- Offline-first (local-first write, sync later)

Tidak menggunakan database server tradisional.

---

15. Data Integrity Rules

- Semua file wajib punya "schema_version"
- Tidak boleh overwrite tanpa backup snapshot
- Merge berdasarkan timestamp terbaru
- Queue wajib diproses saat reconnect
- Tidak ada data loss tanpa recovery attempt

---

16. Workspace Isolation Rule

Setiap workspace:

- memory terpisah
- brain terpisah
- references terpisah
- chats terpisah
- drafts terpisah

Tidak ada cross-workspace leakage kecuali user eksplisit menghubungkan.

---

17. Update v1.1 (Limitless Input)

Batasan `max_length` pada layer validasi Pydantic telah dihapus.
Secara skema, tidak ada batasan panjang karakter di field `content`, `prompt`, atau `message`. Pengguna dapat mengirim dan memproses teks sepanjang apapun selama diizinkan oleh _context window_ model AI.
