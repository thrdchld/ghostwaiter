GhostWriter HF Space Architecture v1.0

1. Visi Produk

GhostWriter adalah aplikasi penulisan berbasis AI yang berjalan di Hugging Face Space, yang berfungsi sebagai sistem pembelajaran gaya menulis dan pola berpikir pengguna.

Tujuan utama:

- Mempelajari gaya menulis pengguna
- Mempelajari cara berpikir pengguna
- Meningkatkan kualitas tulisan berdasarkan data pengguna
- Menggabungkan AI + personal writing system

Bukan sekadar chatbot, tetapi sistem “Writing Intelligence Personal”.

---

2. Prinsip Desain

Sistem ini dibangun dengan prinsip:

- Cepat
- Ringan
- Mobile-first
- Single user
- Workspace-based
- Offline-friendly (partial)
- GitHub-backed storage
- Vanilla frontend (no framework heavy)

4. AI System

Sistem kini mendukung beragam penyedia model AI yang dikonfigurasi melalui Settings:

- OpenRouter (Default Fallback)
- Groq

- Mistral

- Clean generation (tag thinking/chain-of-thought disembunyikan dari UI)

- Writing generation

5. Brain System (Inti AI Personalization)

GhostWriter memiliki 3 profil utama yang dapat dikelola langsung dari panel UI:

5.1 Style Profile
- pilihan kata
- panjang kalimat
- tone
- gaya bahasa

5.2 Thinking Profile
- pola penyelesaian masalah
- struktur kerangka pikir
- sudut pandang argumen

5.3 Memory Profile
- memori percakapan
- fakta-fakta spesifik dari pengguna

Seluruh item di Brain (Style, Thinking, Memory) mendukung Edit dan Hapus secara manual. Tersedia juga fitur Aksi Massal (Setujui Semua / Tolak Semua) untuk usulan AI baru.

8. Workspace System

Setiap workspace adalah environment terisolasi. Mendukung Edit Nama dan Hapus Workspace (kecuali workspace default 'writing' yang dilindungi sistem).

---

9. Storage System

Local Runtime

- File JSON independen (bukan file tunggal raksasa)
- Struktur folder termodulasi (`/system`, `/[workspace_id]/brain`, dll)
- In-memory cache

Sync Storage

- GitHub Repository (sebagai *Source of Truth*)

---

10. Sync System

Sinkronisasi langsung menggunakan *GitHub Git Database API* (Trees & Commits) untuk menjaga struktur folder 1:1 antara lokal dan GitHub. Tidak menggunakan sistem penggabungan timestamp untuk menghindari konflik.

- **Push (Kirim)**: Mem-push seluruh struktur `.json` ke GitHub (overwrite target).
- **Pull (Ambil)**: Menarik seluruh struktur `.json` dari GitHub (overwrite lokal).

Kedua metode memicu Dialog Konfirmasi manual untuk melindungi integritas data pengguna.

18. Update v1.3 (Advanced AI & GitHub Refactor)

Peningkatan yang ditambahkan:
- **Multiple AI Providers**: Support Groq, DeepSeek, Mistral, Gemini, Kilo.
- **True GitHub Mirror**: Sinkronisasi GitHub berbasis Trees yang mem-push folder utuh (bukan 1 JSON file blob) dan mendukung Pull (Restore).
- **Brain Controls**: UI CRUD utuh untuk tab Style, Thinking, dan Memory. Aksi Setujui/Tolak massal.
- **Workspace Protection**: Workspace utama 'writing' kebal hapus, sementara workspace kustom bisa diubah namanya atau dihapus.
- **UI Enhancements**: Penyesuaian nama tab 'Memory', auto-scroll perbaikan, filter `<think>` menyeluruh.
