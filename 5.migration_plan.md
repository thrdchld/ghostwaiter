GhostWriter Migration Plan v1.0

1. Tujuan Migrasi

Mengubah sistem GhostWriter dari:

- Local GGUF-based AI system
- llama.cpp runtime
- Termux/desktop hybrid app

Menjadi:

- Hugging Face Space web app
- Inference API based AI
- Workspace + Brain system
- GitHub-backed persistence

---

2. Komponen yang DIHAPUS TOTAL

Dari sistem lama:

2.1 Model Lokal System

- llama_cpp integration
- GGUF model loading
- RAM management
- device profile (SD660 / Dimensity)
- model upload/download local

---

2.2 Terminal / Agent System

- subprocess CMD execution
- agentic AI terminal access
- OS command simulation
- system shell integration

---

2.3 Local File AI Control

- direct filesystem manipulation by AI
- local model switching via file
- runtime model loading/unloading

---

3. Komponen yang DIPERTAHANKAN (DIREFACTOR)

3.1 Chat System

- tetap ada
- diubah menjadi streaming HF inference
- session disimpan per workspace

---

3.2 Writing System

- prompt → AI → output → edit → train loop
- dihubungkan ke Brain System

---

3.3 Draft System

- tetap ada
- ditingkatkan menjadi:
  - collections
  - tags
  - search full-text

---

3.4 Memory System

- tetap ada
- dipisah per workspace
- diperluas ke Brain System

---

4. SISTEM BARU (WAJIB DIBANGUN)

4.1 Brain System

Menggantikan konsep lama:

- rules saja ❌
- memory saja ❌

Menjadi:

- Style Profile
- Thinking Profile
- Memory
- Rules
- References

---

4.2 Workspace System

Setiap workspace memiliki:

- drafts
- chats
- brain
- references
- learning dataset

Isolasi penuh antar workspace.

---

4.3 Reference System (Internet Learning)

Baru:

- search web (semi-auto)
- user approval required
- store hasil ke references
- reuse sebagai knowledge base

---

4.4 Sync System

- autosave lokal (instant)
- autosync GitHub (30–60 detik)
- manual sync button
- queue fallback jika gagal

8.3 GitHub Sync Conflict

9. OUTPUT AKHIR YANG DIHARAPKAN

GhostWriter versi baru:

- ringan di HP
- cepat di browser
- bisa belajar gaya user
- bisa belajar cara berpikir user
- bisa pakai internet sebagai referensi
- sinkron antar device via GitHub
- tanpa model lokal sama sekali

10. FINAL DECLARATION

Sistem lama = runtime AI lokal
Sistem baru = AI learning + inference cloud

Transformasi ini adalah:

«dari “AI tool” menjadi “AI writing companion yang belajar kamu”»

---

11. Update v1.1 (Pasca-Migrasi Awal)

Penyempurnaan dilakukan pada antarmuka dan kemampuan input:
- **Limitless Mode**: Limit validasi `max_length` dicabut penuh.
- **Mobile First Touch**: _Touch targets_ pada _mobile view_ diperbaiki agar tidak terlalu sempit.
- **Custom Editor Logic**: Fitur Undo kustom (`Ctrl+Z`), *Word count*, dan *Copy chat* diaplikasikan untuk melengkapi _core functionalities_.
